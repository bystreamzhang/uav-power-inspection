# 技术选型文档 · 电力无人机巡检平台

> 已确定：**Vue 3**（前端）· **FastAPI + Python**（后端）· 完全前后端分离
> 本文件讨论其余待定项。

---

## 1. 前端 CSS 框架

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **Tailwind CSS** | 无需命名class、定制灵活、生态强 | 类名冗长、需构建工具 | 定制UI、中大型项目 |
| **Element Plus** | 组件丰富、中文文档、开箱即用 | 风格固定、定制成本高 | 快速交付的企业后台 |
| **Ant Design Vue** | 设计规范、组件完整 | 包体积大、风格偏商务 | 政府/企业系统 |
| **原生 CSS 变量（当前）** | 零依赖、完全可控 | 需自己写所有组件 | 原型/高度定制项目 |

**结论：推荐 Element Plus**
原因：当前项目是电力企业后台，交付周期有压力，Element Plus 的表格、表单、弹窗组件与现有原型页面 1:1 对应，无需从零造轮子，且甲方不在意视觉独特性。

## 2. 前端状态管理

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **Pinia** | Vue 3 官方推荐、轻量、TS支持好 | 无 | Vue 3 项目首选 |
| **Vuex 4** | 老项目兼容 | 模板代码多、已进入维护模式 | 旧项目迁移 |

**结论：Pinia**（无争议）

---

## 3. 2D 地图库

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **Leaflet** | 轻量(42KB)、易上手、插件丰富 | 无原生3D | 航线规划、设备分布、巡检轨迹 |
| **高德地图 JS API 2.0** | 国内卫星图最准确、支持海量点 | 需申请Key、数据在阿里云 | 国内项目定位/路径展示 |
| **MapboxGL JS** | 矢量瓦片、支持2.5D倾斜视角 | 免费额度有限、境外服务 | 美观展示、商业项目 |
| **OpenLayers** | 功能最全、支持WMS/WFS | API复杂、学习成本高 | GIS专业项目 |

**结论：Leaflet + 高德底图瓦片**
Leaflet 加载高德卫星瓦片（免费、无需Key）即可获得准确的国内地图数据，适合航线规划和巡检轨迹展示，工程量小。

## 4. 3D 渲染（WebGL）

> 本项目核心可视化难点，解决方案文档要求三维数字化底座、倾斜摄影建模、点云展示、精细航线规划。

### 4.1 三维地球 / 场景引擎

| 选项 | 底层 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|---|
| **CesiumJS** | WebGL | 开源免费、原生支持点云(.las)、倾斜摄影(3D Tiles)、CZML飞行动画 | 包体积大(>10MB)、学习曲线陡 | **本项目首选**：三维电力线路、杆塔模型、飞行轨迹 |
| **three.js** | WebGL | 极度灵活、生态最大、轻量 | 无地理坐标系、需自己实现GIS功能 | 设备3D模型展示、局部场景 |
| **Babylon.js** | WebGL / WebGPU | 性能强、内置物理引擎 | 地理场景支持弱 | 游戏级3D交互 |
| **deck.gl** | WebGL2 | 擅长大规模地理数据可视化 | 需配合地图底图使用 | 海量点云/轨迹渲染 |

**结论：CesiumJS 作为三维底座**

原因：
- 原生支持 **3D Tiles**（倾斜摄影建模结果的标准格式）
- 原生支持 **点云 .las** 文件加载
- 内置 **CZML** 格式可直接驱动无人机飞行轨迹动画
- 可叠加卫星影像、矢量标注、热力图
- 与 Vue 3 集成成熟（`vue-cesium` 组件库）

### 4.2 辅助：局部设备 3D 模型（杆塔/机库）

| 选项 | 说明 |
|---|---|
| **three.js** | 加载 `.gltf/.glb` 格式杆塔模型，嵌入 Cesium 场景或独立展示 |
| **Cesium 原生 Entity** | 简单几何体（圆柱/线段）模拟杆塔，无需外部模型文件 |

**结论：优先 Cesium Entity，复杂模型用 three.js 嵌入**

### 4.3 渲染层级策略（分阶段）

```
原型阶段   →  SVG 手绘地图（已完成）
一期交付   →  Leaflet 2D + 高德卫星底图（航线规划/设备分布）
二期交付   →  CesiumJS 三维场景（倾斜摄影/点云/飞行轨迹）
远期扩展   →  three.js 局部精细设备模型
```

### 4.4 点云渲染说明

解决方案文档要求展示巡检点云数据（.las 格式），技术路径：

| 环节 | 工具 |
|---|---|
| 后端转换 | Python `laspy` 库读取 .las，转为 Cesium 3D Tiles 格式 |
| 前端渲染 | CesiumJS 原生加载 3D Tiles，支持亿级点云流式加载 |

## 4. 图表库

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **ECharts** | 功能全、中文文档、百度维护 | 包体积较大 | 数据大屏、统计图表 |
| **Chart.js** | 轻量、易用 | 功能相对少 | 简单图表 |
| **D3.js** | 极度灵活 | 学习成本极高 | 高度定制可视化 |

**结论：ECharts**
国内电力行业大屏项目几乎标配，文档和社区资源最丰富。

---

## 5. 后端数据库

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **PostgreSQL** | 功能强、支持JSON、稳定 | 运维略复杂 | 生产环境首选 |
| **SQLite** | 零配置、单文件 | 不适合并发写入 | 本地开发/原型 |
| **MySQL** | 普及率高 | JSON支持弱于PG | 传统企业项目 |

**结论：开发用 SQLite，生产用 PostgreSQL**
FastAPI + SQLAlchemy 切换数据库只需改连接字符串，开发成本为零。

---

## 6. 后端 ORM

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **SQLAlchemy 2.0** | FastAPI 官方推荐、功能完整 | 学习曲线稍陡 | 复杂查询、生产项目 |
| **Tortoise-ORM** | 原生 async、语法简洁 | 生态较小 | 纯异步项目 |

**结论：SQLAlchemy 2.0**（FastAPI 文档默认示例）

---

## 7. 认证方案

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **JWT（本地签发）** | 无状态、简单、无依赖 | Token 无法主动吊销 | 大多数项目 |
| **OAuth2（第三方）** | 支持SSO、企业集成 | 实现复杂 | 需对接企业账号体系 |

**结论：JWT**，FastAPI 内置 OAuth2PasswordBearer 支持，10行代码搞定。

---

## 8. 实时通信（无人机状态推送）

| 选项 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| **WebSocket** | 双向实时、FastAPI 原生支持 | 需维护连接状态 | 无人机实时状态/视频帧 |
| **SSE（Server-Sent Events）** | 简单、单向推送 | 仅服务端→客户端 | 任务进度通知 |
| **轮询** | 实现最简单 | 延迟高、浪费资源 | 不推荐 |

**结论：WebSocket（实时状态）+ SSE（任务通知）**
